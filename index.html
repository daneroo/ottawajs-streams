What are streams

We should have some ways of connecting programs like garden hose--screw in another segment when it becomes necessary to massage data in another way. This is the way of IO also.
--Doug Mcilroy '64
  • Modeled after Unix I/O
  • Used to process data in stages

          grep daniel README.md |sed -e "s/daniel/daneroo/g"

Examples

  • fs.ReadStream, fs.WriteStream
  • tcp/tls Socket objects
  • http request/response
  • zlib, stdio, etc

Updates in node 0.10

  • referred to as streams2
  • Compatible 0.8<-->0.10
  • Simpler API
  • Used for core fs/zlib/crypto/http/tls/tcp
  • Consistent behavior (pause(), lost 'data')
  • Buffering and backpressure implementation

Kinds of streams

  • Readable
  • Writeable
  • Transform
  • Duplex
  • Passthrough

Readable

  • inherit from Readable
  • implement _read(size)
  • call push()

Writeable

  • inherit from Writeable
  • implement _write(chunk, encoding, callback)
  • invoke callback()
